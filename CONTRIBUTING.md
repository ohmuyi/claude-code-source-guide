# 贡献指南

这份文档用于说明如何维护《Claude Code源码精讲》这套 Markdown 书稿，使其长期适合 GitHub 阅读与协作。

## 基本原则

- `chapters/` 是唯一正文来源
- `README.md` 只负责仓库首页与阅读引导
- `SUMMARY.md` 负责简洁目录
- `Claude-Code源码精讲.md` 负责章节导航页
- 修改章节时，优先保持文件名稳定，避免破坏外部链接

## 推荐修改流程

1. 先确定你要修改的是哪一章  
如果只是正文修订，只改对应章节文件。

2. 如果新增章节或调整顺序  
同时更新 `SUMMARY.md`、`Claude-Code源码精讲.md`、`README.md` 中的目录信息。

3. 如果重命名章节文件  
同步更新所有引用它的导航链接，包括相邻章节的“上一章 / 下一章”。

## 章节格式约定

每章保持同样的结构：

1. `# 第X章 标题`
2. `## 本章要回答的问题`
3. `## 关键源码入口`
4. `## 运行机制`
5. `## 设计权衡`
6. `## 小结`
7. `## 导航`

这样做是为了让读者在任何一章都能快速建立预期，也便于不同贡献者协作。

## 写作约定

- 默认使用简体中文
- 代码、命令、路径、类型名保留原文
- 不做逐文件流水账，优先解释系统边界、调用链和设计权衡
- GitHub 文档优先短段落，避免大块正文
- 单章内优先使用相对链接，方便仓库迁移

## 提交建议

- 小改动按章节提交，避免一个 PR 混杂多章
- 标题调整与正文大改尽量分开提交
- 如果改动会影响目录结构，请在 PR 描述里明确说明

## 不建议做的事情

- 不要重新引入单文件合订本作为人工维护文件
- 不要频繁改章节文件名
- 不要把首页 `README.md` 写成正文，首页应保持导航功能
