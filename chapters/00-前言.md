# 前言

很多人第一次打开 Claude Code 这类项目时，都会产生一个错觉：这不过是一个命令行外壳，外面包着几个工具调用，再加上一些 UI 组件。可是真正把源码向下读几层之后，你会发现它更像一个运行时系统。它要处理启动分流、状态同步、消息流、工具编排、权限决策、后台任务、远程会话、扩展生态，以及多 agent 协同带来的复杂边界。

因此，这本《Claude Code源码精讲》不采用“按目录扫一遍”的写法，而采用“先建立世界观，再解释机制”的写法。这个组织方式借鉴《C++编程思想》的优点：先讲对象和模型，再讲行为和约束，最后讲扩展和演进。对读者而言，这样的路径比直接追调用栈更稳定，也更接近工程师真正理解系统的方式。

本书默认你具备 TypeScript、React、Node.js 和命令行程序的基础，但不要求你事先理解 Claude Code 的内部协议。我们会把它当成一个完整产品来分析：它的核心抽象是什么，它如何把模型输出变成工具执行，它如何在终端里组织交互体验，它又如何把本地运行时延伸成远程和分布式协作平台。

阅读这本书时，请始终记住一个核心判断：Claude Code 的难点并不在某个孤立算法，而在多个稳定抽象之间的组合。只要你抓住 `QueryEngine`、`Tool`、`Task`、`AppState`、`REPL`、`MCP`、`RemoteSession` 这几条主轴，剩余的大量代码就会逐步退化为实现细节，而不再是一片没有边界的森林。
